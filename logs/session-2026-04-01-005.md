  **How dashboard prompt execution works end-to-end:**
  1. User submits prompt via dashboard → POST /api/projects/{id}/prompt
  2. poll_prompts.py picks it up in next 60s poll cycle
  3. Creates CPO task directory in ~/dev/queue/pending/ with full task.md
  4. Automatically runs queue-worker-full-pass.sh
  5. Claude Code reads task.md, executes the objective fully
  6. Writes response to thread.jsonl, commits changes
  7. poll_prompts.py logs success/failure, posts SSE event

  **How to mark a WORKQUEUE task as overnight-ready:**
  In WORKQUEUE-MASTER.md, add these fields to a P1 task:
  ```
  overnight-ready: YES
  blast-radius: LOW  (or NONE)
  project: <project-id>
  ```
  The overnight timer only processes P1 tasks. Move tasks to P1 before
  the overnight run.

  **Blast radius enforcement in overnight.py:**
  - NONE/LOW: executes automatically
  - MEDIUM/HIGH/UNKNOWN: skipped, ntfy notification sent, NOTIFY.md entry written
  - If task encounters unexpected blast-radius during execution, task.md
    instructs Claude Code to STOP and write to NOTIFY.md
